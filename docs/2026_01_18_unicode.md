**Created_at:** 2026-01-18  
**Updated_at:** 2026-03-12  
**Status:** Active  
**Goal:** Reference for Unicode box-drawing characters used in Sketch Pad and ASCII diagrams  
**Scope:** Box-drawing characters, combining marks, Sketch Pad rendering

## Rationale

Zen IDE uses Unicode box-drawing characters (U+2500–U+257F) instead of plain ASCII (`+`, `-`, `|`) for three reasons:

1. **Visual clarity** — Box-drawing characters form seamless connected lines with no gaps at intersections. A rectangle drawn with `┌─┐│└┘` reads as a single shape, while `+-+|` reads as disconnected punctuation.
2. **Semantic precision** — Each character encodes its connectivity (e.g. `┤` connects top, bottom, and left but not right). This lets the Sketch Pad select the correct character automatically when two shapes share an edge, and allows direction-aware corner routing for arrows.
3. **Monospace alignment** — Box-drawing characters occupy exactly one cell in monospace fonts, preserving the grid-based layout that ASCII art depends on.

### Clipboard fallback

Some terminals and editors assign East Asian Width "Ambiguous" to box-drawing characters, rendering them double-wide and breaking alignment. To handle this, `sketch_model.py` includes a translation table (`_CLIPBOARD_XLAT`) that maps Unicode box-drawing characters to plain ASCII equivalents when copying to the clipboard:

```
┌┐└┘─│┬┴├┤╭╮╯╰▲▼►◄  →  ++++-|++++++++^v><
```

This ensures pasted diagrams remain readable in any environment.

## Character Set

### Standard Box-Drawing (U+2500 block)

| Character | Codepoint | Name                          | Usage in Zen                     |
|-----------|-----------|-------------------------------|----------------------------------|
| `─`       | U+2500    | Light horizontal              | Horizontal edges, arrow segments |
| `│`       | U+2502    | Light vertical                | Vertical edges, arrow segments   |
| `┌`       | U+250C    | Light down and right          | Top-left corner                  |
| `┐`       | U+2510    | Light down and left           | Top-right corner                 |
| `└`       | U+2514    | Light up and right            | Bottom-left corner               |
| `┘`       | U+2518    | Light up and left             | Bottom-right corner              |
| `├`       | U+251C    | Light vertical and right      | Left T-junction (Database shape) |
| `┤`       | U+2524    | Light vertical and left       | Right T-junction (Database shape)|
| `┬`       | U+252C    | Light down and horizontal     | Top T-junction (Topic divider)   |
| `┴`       | U+2534    | Light up and horizontal       | Bottom T-junction (Topic divider)|
| `┼`       | U+253C    | Light vertical and horizontal | Cross-junction (tables)          |

### Rounded Corners (U+256D–U+2570)

| Character | Codepoint | Name                    | Usage in Zen        |
|-----------|-----------|-------------------------|---------------------|
| `╭`       | U+256D    | Light arc down and right| Database top-left   |
| `╮`       | U+256E    | Light arc down and left | Database top-right  |
| `╰`       | U+2570    | Light arc up and right  | Database bottom-left|
| `╯`       | U+256F    | Light arc up and left   | Database bottom-right|

### Arrow Heads (Block Elements / Geometric Shapes)

| Character | Codepoint | Name                          | Direction |
|-----------|-----------|-------------------------------|-----------|
| `▲`       | U+25B2    | Black up-pointing triangle    | Up        |
| `▼`       | U+25BC    | Black down-pointing triangle  | Down      |
| `►`       | U+25BA    | Black right-pointing pointer  | Right     |
| `◄`       | U+25C4    | Black left-pointing pointer   | Left      |

## Usage by Shape

| Shape        | Characters Used        | Notes                                          |
|--------------|------------------------|-------------------------------------------------|
| Rectangle    | `┌ ┐ └ ┘ ─ │`         | Standard box border via `BorderChars`           |
| Topic        | `┌ ┐ └ ┘ ─ │ ┬ ┴`     | Box with internal horizontal dividers           |
| Database     | `╭ ╮ ╰ ╯ ─ │ ├ ┤`     | Cylinder shape using rounded corners            |
| Arrow        | `─ │ ┌ ┐ └ ┘ ▲ ▼ ► ◄` | L-shaped routing with direction-aware corners   |
| Cloud        | `( ) _ .`              | ASCII-only, no box-drawing characters           |

## Corner Selection Logic

Arrow corners are selected automatically from the `CORNER_CHARS` lookup table based on the incoming and outgoing direction of the path:

```
(incoming, outgoing) → character

(right, down)  → ┐     (left, down)  → ┌
(right, up)    → ┘     (left, up)    → └
(down, right)  → └     (up, right)   → ┌
(down, left)   → ┘     (up, left)    → ┐
```

This eliminates manual corner selection and ensures correct rendering for any L-shaped arrow path.

## Other Uses in Zen

- **Terminal markdown renderer** — Code block borders (`┌─┐│└┘`) and table separators (`├┼┤┴`).
- **Markdown formatter** — Decorative horizontal rules (`─` repeated).
- **Tree view** — Comments reference `└` and `│` for tree-line logic (actual rendering is graphical).
