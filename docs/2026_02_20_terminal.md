Zen IDE includes a fully integrated terminal powered by **VTE** (Virtual Terminal Emulator) via the `Vte.Terminal` widget. It lives in the bottom-right panel alongside the AI chat.

## Architecture

| Component | File | Role |
|-----------|------|------|
| `TerminalStack` | `src/terminal/terminal_stack.py` | Container managing terminal panes (vertical stack or tabbed) |
| `TerminalView` | `src/terminal/terminal_view.py` | Single terminal pane (VTE + header UI) |
| `TerminalHeader` | `src/terminal/terminal_header.py` | Shared header bar builder for terminal panes |
| `TerminalTabButton` | `src/terminal/terminal_tab_button.py` | Tab button widget for tab bar mode |
| `TerminalShellMixin` | `src/terminal/terminal_shell.py` | Shell spawning, custom bashrc, directory navigation |
| `TerminalShortcutsMixin` | `src/terminal/terminal_shortcuts.py` | Copy/paste, word navigation, context menu |
| `TerminalFileNavigationMixin` | `src/terminal/terminal_file_navigation.py` | File path detection, Cmd+click navigation |
| `terminal_scroll` | `src/terminal/terminal_scroll.py` | VTE scrolling configuration and scroll delta mapping |
| `vte_compat` | `src/terminal/vte_compat.py` | VTE availability detection and import shim |

`TerminalStack` extends `FocusBorderMixin` and `Gtk.Box` and holds one or more `TerminalView` instances. It supports two layout modes controlled by the `behavior.terminals_on_vertical_stack` setting:

- **Vertical stack** (default, `true`): all terminals are stacked vertically and visible simultaneously.
- **Tab bar** (`false`): a horizontal scrollable tab bar sits at the top and only the active terminal is visible (like the AI chat panel).

`TerminalView` extends `Gtk.Box` with `TerminalFileNavigationMixin`, `TerminalShellMixin`, and `TerminalShortcutsMixin`. It wraps a `Vte.Terminal` inside a scrolled window with a header bar on top.

## UI Layout

```
┌──────────────────────────────────────────┐
│  TERMINAL (clickable)  [+] [clear] [⛶]  │  ← header bar
├──────────────────────────────────────────┤
│                                          │
│  VTE terminal (bash shell)               │
│                                          │
└──────────────────────────────────────────┘
```

- **TERMINAL label** — clickable button; opens a workspace project picker to `cd` into a different workspace folder.
- **+ button** — splits the terminal panel, adding a new terminal pane below (vertical stack).
- **Clear button** — clears the terminal by running `clear` and flushing the VTE scrollback buffer via the ED3 escape sequence (`\033[3J`).
- **Maximize button (⛶)** — toggles maximizing the terminal panel to fill the window.
- **× close button** — closes this terminal pane (only visible when multiple panes exist).

### Terminal Stack (Vertical Mode — default)

Clicking the **+** button creates a new terminal pane below the current one. Each pane is an independent `TerminalView` with its own shell, cwd, and header. The stack is managed by `TerminalStack`.

```
┌──────────────────────────────────────────┐
│  TERMINAL        [+] [clear] [⛶]        │  ← terminal 1 header
├──────────────────────────────────────────┤
│  VTE terminal 1                          │
├──────────────────────────────────────────┤
│  TERMINAL        [+] [clear] [×] [⛶]    │  ← terminal 2 header
├──────────────────────────────────────────┤
│  VTE terminal 2                          │
└──────────────────────────────────────────┘
```

Each stacked terminal:
- Has its own shell process and working directory
- Can be independently cleared, cd'd to a different project, or closed
- The **×** close button only appears when there are 2+ terminal panes
- New terminals inherit the cwd of the currently active terminal

### Terminal Tabs (Tab Bar Mode)

When `behavior.terminals_on_vertical_stack` is `false`, terminals are displayed one at a time with a horizontal scrollable tab bar at the top (similar to the AI chat panel).

```
┌──────────────────────────────────────────┐
│ [Terminal 1 ×] [Terminal 2 ×]            │  ← scrollable tab bar
├──────────────────────────────────────────┤
│  TERMINAL        [clear] [⛶]            │  ← active terminal header
├──────────────────────────────────────────┤
│                                          │
│  VTE terminal (active tab)               │
│                                          │
└──────────────────────────────────────────┘
```

In tab mode:
- Clicking a tab switches the visible terminal
- Tab close buttons (×) appear when there are 2+ tabs
- The per-terminal **+** and **×** buttons are hidden; use the tab bar instead
- Each terminal still has its own shell, cwd, clear, and maximize buttons

## Shell Environment

The terminal spawns a **bash** shell with a custom `~/.zen_ide/bashrc` that:

1. Sources the user's `~/.bash_profile` or `~/.bashrc`
2. Sets color environment variables (`TERM=xterm-256color`, `COLORTERM=truecolor`, `FORCE_COLOR=1`, etc.)
3. Configures readline bindings (tab completion, case-insensitive completion, history search with arrow keys)
4. Persists history to `~/.zen_ide/bash_history` (1000 lines), while suppressing startup/bootstrap noise and known AI probe commands from history
5. Defines built-in aliases (see [terminal_aliases.md](2026_02_18_terminal_aliases.md))
6. Sources user custom aliases from `~/.zen_ide/aliases`
7. Sets a git-aware prompt showing `directory (branch *)` with dirty-state indicator

## File Path Navigation (Cmd+Click)

The terminal detects file paths in output and underlines them on hover. `Cmd+Click` (or `Ctrl+Click` on Linux) opens the file in the editor.

Matching is done in two layers:

1. **VTE regex match** — a PCRE2 pattern registered with `match_add_regex` that underlines paths on hover and sets a pointer cursor.
2. **Fallback text extraction** — if VTE match fails, the terminal manually reads the row text and applies a Python regex (`FILE_PATH_PATTERN`) to find paths near the click position.

Supported patterns:
- Relative paths: `src/main.py`, `./lib/utils.js`
- Absolute paths: `/usr/local/bin/script.sh`
- With line numbers: `file.py:42`, `file.py:42:10`
- Known extensionless files: `Makefile`, `Dockerfile`, `Gemfile`, etc.

Relative paths are resolved against the terminal CWD and workspace folders.

## Terminal Follow File

When `behavior.terminal_follow_file` is `true` (default), switching to a file in the editor automatically `cd`s the terminal to that file's git repository root. If this navigation crosses repository boundaries, the terminal is also auto-cleared to avoid mixed output from different repos.

## Debounced Resize

VTE keeps text rewrapping enabled (`set_rewrap_on_resize(True)`) so wrapped
prompt lines are correctly reflowed when the pane is shrunk and expanded again.

## Theming

Colors are applied from the active theme:

| Property | Source |
|----------|--------|
| Foreground | `theme.fg_color` |
| Background | `theme.panel_bg` |
| ANSI palette (16 colors) | `theme.term_black` through `theme.term_white` + bright variants (auto-lightened by 20%) |

Theme changes are applied live via `subscribe_theme_change`.

## Font Configuration

Terminal font is configured via `fonts.terminal` settings:

| Setting | Default | Description |
|---------|---------|-------------|
| `terminal.font_family` | Platform default (e.g. Menlo on macOS) | Font family name |
| `terminal.font_size` | `12` | Font size in points |
| `terminal.font_weight` | `"normal"` | Font weight |

VTE requires true monospace fonts. The `_sanitize_font_for_vte()` function automatically converts "Propo" (proportional) Nerd Font variants to their "Mono" equivalents.

## Settings Reference

| Setting | Default | Description |
|---------|---------|-------------|
| `terminal.font_family` | `""` (platform default) | Terminal font family |
| `terminal.font_size` | `12` | Terminal font size |
| `terminal.scrollback_lines` | `10000` | Number of scrollback lines |
| `scroll_speed` | `0.4` | Global scroll speed multiplier (affects terminal wheel/touchpad step size) |
| `cursor_blink` | `true` | Enable cursor blink (applies to terminal and editor) |
| `wide_cursor` | `false` | Use block cursor instead of I-beam in terminal |

| `behavior.terminals_on_vertical_stack` | `true` | `true` = vertical split panes, `false` = horizontal tab bar |

## Scrolling Behavior

Terminal scrolling uses a lerp-based smooth animation on top of VTE's pixel scroll
support:

- `Gtk.ScrolledWindow` has kinetic scrolling enabled for trackpad momentum and
  deceleration.
- `Vte.Terminal` enables fallback scrolling so unconsumed wheel/trackpad events
  continue into the parent `Gtk.ScrolledWindow`.
- `Vte.Terminal` is configured with pixel scroll units when supported
  (`scroll-unit-is-pixels`), so the exposed `GtkAdjustment` uses pixels instead
  of terminal rows.
- Zen installs a scroll-event controller in **capture** phase that maps both
  wheel and touchpad deltas to pixel values.  Discrete wheel ticks use
  `15 × scroll_speed` pixels per notch; touchpad events use
  `12 × scroll_speed` pixels per delta (default `scroll_speed` is `0.4`).
- Instead of jumping to the new position, deltas are **accumulated into a
  target** and an `add_tick_callback` lerp animates the viewport toward it
  each frame (30 % of remaining distance per frame).  This produces
  sub-row smooth scrolling even though VTE renders a row-aligned grid.
- Kinetic/momentum events are handled through the GTK scroll-controller
  kinetic flag, so deceleration remains smooth after trackpad release.

## Lifecycle

1. **Lazy creation** — the terminal panel starts as a placeholder `Gtk.Box`. The real `TerminalStack` (containing a `TerminalView`) is created in `_create_bottom_panels()` (deferred for faster first paint).
2. **Shell spawn** — `spawn_shell()` is called after the UI is ready; it uses `Vte.Terminal.spawn_async` with the custom bashrc.
3. **Auto-respawn** — if the shell exits (and the IDE isn't shutting down), a new shell is spawned after 100ms.
4. **Cleanup** — on IDE shutdown, `cleanup()` sets a shutdown flag and kills the shell process group via `SIGTERM`.

## Context Menu

Right-click on the terminal opens a context menu with:
- **Copy** — copy selected text
- **Paste** — paste from clipboard
- **Clear** — clear the terminal

## Workspace Project Picker

Clicking the "TERMINAL" header label opens an `NvimContextMenu` listing all workspace folders. The current folder is marked with ✓. Selecting a folder runs `cd` in the terminal.

## Fallback Mode

If VTE is not available (missing `gi.repository.Vte`), the terminal falls back to a read-only `Gtk.TextView` with a message to install VTE. No shell is spawned in this mode.
