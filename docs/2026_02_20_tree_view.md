# Tree View

**Created_at:** 2026-02-20  
**Updated_at:** 2026-03-16  
**Status:** Active  
**Goal:** Document the file explorer tree with Nerd Font icons, git status badges, and drag-and-drop  
**Scope:** `src/treeview/tree_view.py`, keyboard navigation, context menus  

---

Zen IDE's file explorer is a custom Cairo-drawn tree panel with Neovim-style navigation, git integration, and drag-and-drop support.

## Keyboard Navigation

| Key | Action |
|-----|--------|
| `Cmd+Shift+E` / `Ctrl+Shift+E` | Focus tree explorer |
| `Cmd+P` / `Ctrl+P` | Quick open file (fuzzy finder) |
| `Ōåæ` / `k` | Move selection up |
| `Ōåō` / `j` | Move selection down |
| `ŌåÆ` / `l` / `Enter` | Expand folder or open file |
| `ŌåÉ` / `h` | Collapse folder or go to parent |
| `Cmd+Click` / `Ctrl+Click` | Toggle a row into multi-selection |
| `Shift+Click` | Select the full range from the anchor row to the clicked row |
| `Cmd+Delete` / `Ctrl+Delete` | Delete the current selection after confirmation |
| `Cmd+C` / `Ctrl+C` | Copy all selected tree items |
| `Cmd+V` / `Ctrl+V` | Paste all copied tree items into the current target |

## File Icons

Icons are rendered using **Nerd Fonts** when available, with emoji fallback otherwise.

Over 50 file types are recognized with unique icons and colors:

| Type | Color | Examples |
|------|-------|----------|
| Python | `#ffbc03` | `.py` |
| TypeScript | `#3178c6` | `.ts`, `.tsx` |
| JavaScript | `#f1e05a` | `.js`, `.jsx` |
| Rust | `#dea584` | `.rs` |
| Go | `#00add8` | `.go` |
| HTML | `#e44d26` | `.html` |
| CSS | `#563d7c` | `.css` |
| Markdown | `#519aba` | `.md` |
| Docker | `#2496ed` | `Dockerfile` |
| Config | Various | `Makefile`, `.gitignore`, `package.json`, etc. |

Folders use a dedicated folder icon that changes between open/closed states.

### Nerd Font Detection

On first run, Zen scans for installed Nerd Fonts (JetBrains Mono, FiraCode, Hack, CaskaydiaCove, etc.) and caches the result in `~/.zen_ide/font_cache.txt`. If no Nerd Font is found, emoji icons are used as fallback.

## Git Status Indicators

Files and folders display colored status badges showing their git state:

| Badge | Meaning | Color |
|-------|---------|-------|
| `M` | Modified | Yellow (`#e5c07b`) |
| `A` | Added | Green (`#98c379`) |
| `D` | Deleted | Red (`#e06c75`) |
| `R` | Renamed | Blue (`#61afef`) |
| `?` | Untracked | Bright green (`#73c991`) |
| `I` | Gitignored | Dim gray |

Parent directories inherit the modified color when any child has changes, even if the folder is collapsed.

## Gitignore Filtering

The tree respects `.gitignore` patterns using the `GitIgnoreUtils` module. Patterns are compiled to regex and cached per workspace root for performance. Both global and workspace-scoped patterns are supported.

## Context Menu

Right-click any item to access file operations.

When multiple rows are selected, the context menu applies bulk actions to the full selection. Nested descendants under a selected folder are de-duplicated so copy and delete only act once on the top-level selection.

### File Actions

| Action | Description |
|--------|-------------|
| Open | Open file in editor |
| Show in Finder | Reveal in system file manager |
| Copy Path | Copy full path to clipboard |
| Rename | Inline rename with extension preserved |
| Delete | Delete with confirmation |
| Discard Local Changes | Git revert to HEAD (shown for modified files) |

### Folder Actions

| Action | Description |
|--------|-------------|
| New File | Create file with inline naming |
| New Folder | Create folder with inline naming |
| Show in Finder | Open in system file manager |
| Copy Path | Copy full path to clipboard |
| Rename | Inline rename |
| Delete | Recursive deletion |

## Inline Editing

Creating or renaming files happens directly in the tree via an inline text entry:

- **New file/folder**: A placeholder row appears; type the name and press `Enter` to confirm, `Escape` to cancel.
- **Rename**: The existing name opens in edit mode. For files, only the name (without extension) is selected by default.
- Newly created files are automatically opened in the editor.

## Drag & Drop

Files and folders can be moved by dragging them within the tree.

### Drop Positions

- **On a folder** (middle 50%): moves the item into that folder
- **Before/after** (top/bottom 25%): reorders at that position
- **On a file** (top/bottom 50%): inserts before or after

### Validation Rules

- Cannot drop an item onto itself
- Cannot drop a folder into its own descendants
- Cannot drop to the same parent at the same position
- Rejected if a file with the same name already exists at the destination

A visual indicator (highlighted row or line between rows) shows where the item will be placed during drag.

## Indent Guides

Neovim-style vertical indent guides show the tree hierarchy:

- `Ōöé` continuation lines for sibling items
- `Ōöö` corner connectors for last children
- 16px indentation per depth level

## Smooth Scrolling

- Keyboard navigation auto-scrolls to keep the selection visible
- Animated reveal uses cubic ease-out for smooth deceleration
- Kinetic scrolling is supported via GTK4 trackpad momentum

## Workspace Support

### Multiple Folders

The tree supports multiple workspace roots. Each root appears as a top-level entry.

### Refresh Behavior

When the tree refreshes (e.g., after file operations), it preserves:

- Expanded folder state
- Current selection
- Scroll position

### Reveal File

When you switch tabs in the editor, the tree automatically expands parent folders and scrolls to reveal the active file.

## Configuration

Tree behavior is controlled by constants in `src/constants.py` and settings in `~/.zen_ide/settings.json`:

**Settings (`~/.zen_ide/settings.json`):**

| Setting | Default | Description |
|---------|---------|-------------|
| `treeview.line_spacing` | `10` | Total vertical spacing (px) split evenly above/below each row |

**Constants (`src/constants.py`):**

| Constant | Default | Description |
|----------|---------|-------------|
| `DEFAULT_TREE_WIDTH` | `400` | Sidebar width in pixels |
| `DEFAULT_TREE_MIN_WIDTH` | `0` | Minimum resize width |
| `TREE_SCROLL_SPEED` | `0.15` | Rows per scroll notch |

## Performance

- **Lazy loading**: children are loaded only when a folder is expanded
- **Virtual scrolling**: only visible rows are rendered
- **Redraw coalescing**: multiple redraws are batched into a single frame
- **Pattern caching**: gitignore patterns are compiled once per workspace
- **Font caching**: Nerd Font detection result cached to disk
