# Icons — Use the Icon Manager

**Created_at:** 2026-02-19  
**Updated_at:** 2026-03-16  
**Status:** Active  
**Goal:** Enforce centralised icon management — no hardcoded Unicode code-points  
**Scope:** `src/icons/icon_manager.py`, all UI components using icons  

---

## Rule

**All icons must be defined in `src/icons/icon_manager.py` and imported via `from icons import Icons`.** Never hardcode Unicode code-points directly in components.

When adding icons to the UI, prefer [Nerd Font](https://www.nerdfonts.com/cheat-sheet) icons. Only fall back to other sources (e.g., GTK/Adwaita icons) if no suitable Nerd Font icon exists.

```python
# BAD - hardcoded icon character
btn = Gtk.Button(label="\uf067")

# GOOD - centralised constant
from icons import Icons
btn = Gtk.Button(label=Icons.PLUS)
```

## Two-Font Architecture

Zen IDE uses exactly **two fonts**:

| Font | File | Purpose |
|------|------|---------|
| **ZenIcons** | `src/fonts/resources/ZenIcons.ttf` (~40 KB) | All icon glyphs — Nerd Font subset |
| **Source Code Pro** | `src/fonts/resources/SourceCodePro-VariableFont_wght.ttf` (~204 KB) | All text (editor, terminal, UI) |

Both fonts are registered at startup via `_register_fonts_early()` in `zen_ide.py`.

**ZenIcons** is generated by `tools/subset_icon_font.py` — it subsets the full SauceCodePro Nerd Font to only the codepoints used by `Icons.*` constants and tree icons, reducing ~3 MB to ~40 KB.

Every icon label must render with the `ZenIcons` font family. Use the centralised helpers to ensure this.

## Icon Manager API

| Function / Constant | Description |
|---------------------|-------------|
| `Icons.*` | Named constants for every icon (actions, notifications, file types, sketch tools) |
| `ICON_FONT_FAMILY` | `"ZenIcons"` — the bundled icon font family name |
| `get_icon_font_name()` | Returns `ICON_FONT_FAMILY` (replaces old `get_nerd_font_name()`) |
| `create_icon_label(icon, size_pt)` | Creates a `Gtk.Label` with ZenIcons font and consistent sizing |
| `apply_icon_font(label, size_pt)` | Sets ZenIcons font on an existing `Gtk.Label` via Pango attributes |
| `icon_font_css_rule()` | Returns a CSS snippet setting icon font on `.zen-icon` labels |
| `get_file_icon(path)` | Returns `(icon_char, hex_color)` for a file path |
| `ICON_SIZE_CSS_CLASS` | `"zen-icon"` — CSS class applied to all icon labels |

### Usage Examples

```python
# Create an icon label (preferred for standalone icons)
from icons import Icons, create_icon_label
label = create_icon_label(Icons.PLUS)

# Set icon font on an existing label
from icons import Icons, apply_icon_font
btn = Gtk.Button(label=Icons.CLOSE)
apply_icon_font(btn.get_child())

# Use icon font in CSS
from icons import get_icon_font_name
css = f'.my-icon {{ font-family: "{get_icon_font_name()}"; }}'

# Use the CSS rule helper
from icons import icon_font_css_rule
css = f'''
    {icon_font_css_rule()}
    .my-widget {{ color: red; }}
'''
```

- `treeview.tree_icons.get_nerd_font_name()` — still works but delegates to `icons.get_icon_font_name()`. Use the `icons` import instead.
