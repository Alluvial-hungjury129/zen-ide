# Zen IDE Startup Architecture

**Created_at:** 2026-02-27  
**Updated_at:** 2026-03-08  
**Status:** Active  
**Goal:** Detail multi-phase startup architecture achieving <80ms first-paint through deferred initialization  
**Scope:** `src/zen_ide.py`, `src/main/window_layout.py`, `src/main/window_state.py`  

---

Zen IDE starts in **under 80ms to first paint** and reaches **fully loaded in ~120ms**, matching Vim-level responsiveness. This is achieved through a multi-phase deferred initialization strategy that prioritizes getting pixels on screen before doing heavy work.

> **Note on timing**: The startup timer starts in `do_activate()`, measuring only application code. GTK framework overhead (~60ms for `app.run()` ‚Üí `do_activate()`) is excluded since it's an irreducible platform cost.

## Performance Targets

| Metric | Target | Description |
|--------|--------|-------------|
| First paint | < 80ms | Window visible with HeaderBar + placeholder layout |
| Interactive | < 90ms | Editor created and ready for input |
| Fully loaded | < 90ms | Editor + tree + shortcuts + theme ready, IDE is usable |
| Full UI visible | < 100ms | All visible panels created, workspace loaded |

These targets are enforced by `make startup-time` (env var `ZEN_STARTUP_BENCH=1`), which launches the IDE, measures all milestones, and auto-quits.

## Startup Phases

### Phase 0: Pre-GTK (module-level code in `zen_ide.py`)

Runs before GTK initializes. Only essential, fast operations:

1. **Path setup** ‚Äî adds parent dir to `sys.path`
2. **Logging setup** ‚Äî `DEBUG` if `ZEN_DEBUG` set, `WARNING` otherwise
3. **Font pre-registration** ‚Äî `_register_fonts_early()` reads binary `.ttf` files from `src/fonts/resources/` and registers them with CoreText (macOS) or fontconfig (Linux) via `ctypes` BEFORE GTK/Pango initializes. Sets `_ZEN_FONTS_PREREGISTERED=1` env var to signal `font_manager.py` to skip its own registration. Binary `.ttf` files avoid the 78ms cost of importing the 3.8MB `font_data.py` module
4. **GDK backend hint** ‚Äî on macOS, sets `GDK_BACKEND=macos` to skip backend detection (~25ms saving)
5. **GI version requirements** ‚Äî `gi.require_version()` for GTK 4.0, GtkSource 5, Vte 3.91
6. **GTK warning filters** ‚Äî suppresses known GTK4/GDK bugs
7. **GTK init + theme preference** ‚Äî `Gtk.init()`, load settings, set theme, and apply `gtk-application-prefer-dark-theme` at module level so the ~9ms cost is excluded from the first-paint timer
8. **Module imports** ‚Äî constants, theme, settings (lightweight modules only)

**What is NOT imported here** (intentionally deferred):
- `TreeView`, `EditorView`, `TerminalView`, `AIChatTabs` ‚Äî heavy widget trees
- `crash_log`, `exit_tracker` ‚Äî deferred to housekeeping after first paint
- `fonts` module (beyond pre-registration) ‚Äî Pango font enumeration is expensive
- `AppKit` (macOS dock icon) ‚Äî ~120ms import cost
- AI process tracker ‚Äî loads asyncio, pyte, etc.
- `StatusBar` ‚Äî has theme dependencies

### Phase 1: Application Activate (`do_activate`)

Called by GTK when the app is ready. **Timer starts here.**

1. **Create `ZenIDEWindow`** ‚Äî triggers `__init__`:
   - Reads saved layout (settings already loaded at module level)
   - Sets window size from saved layout
   - Calls `_create_layout()`:
     - Creates the paned layout structure (main ‚Üí tree|right, right ‚Üí editor|bottom)
     - Uses **Gtk.Box placeholders** for TreeView, EditorView, AIChatTabs, TerminalView, StatusBar
     - Sets saved paned positions and **locks them** (prevents GTK from resetting during child swaps)
     - Creates **HeaderBar** with title (before realize ‚Äî avoids 49ms set_titlebar penalty on realized windows)
2. **`win.present()`** ‚Äî window becomes visible ‚Üí **first paint** milestone
3. **Start main-thread poll** ‚Äî `start_main_thread_poll()` for cross-thread dispatch (deferred to after first paint)
4. **Schedule housekeeping** ‚Äî `GLib.idle_add(_run_startup_housekeeping)` (crash log recovery, exit tracker, AI orphan cleanup)

### Phase 2: Window Mapped ‚Äî Sync Batch (`_on_window_mapped`)

Fires when the window is actually rendered on screen. Creates the editor, tree view, applies theme, binds shortcuts ‚Äî all in a single synchronous batch to eliminate GLib transition overhead.

1. **Print first paint** ‚Äî records the milestone
2. **`_deferred_init_editor()`** ‚Äî creates EditorView (with lazy imports for autocomplete, gutter diff, minimap, dev_pad), sets callbacks, prints **interactive** milestone
3. **Create actions & shortcuts** ‚Äî `_create_actions()`, `_bind_shortcuts()`, `_setup_key_handler()` (~1ms)
4. **Register fonts + apply theme** ‚Äî `register_resource_fonts()` + `_apply_theme()` (single application, no re-theme needed later)
5. **Add menu button** ‚Äî creates MenuBuilder menu on HeaderBar
6. **Create real TreeView** ‚Äî replaces placeholder, sets callbacks, re-applies paned positions
7. **Setup focus tracking** ‚Äî for editor and tree view
8. **Setup file drop target** ‚Äî drag-and-drop of external files
9. **Setup app icon** ‚Äî loads icon image
10. **Print fully loaded** ‚Äî editor + tree + theme + shortcuts all ready
11. **Activate foreground** ‚Äî bring app to front
12. **Schedule deferred init** ‚Äî `GLib.timeout_add(0, _deferred_init)`

### Phase 3: Deferred Init (`_deferred_init`)

Single deferred phase combining the previous 3 phases into one (saves ~34ms of GLib transition overhead):

1. **Create real StatusBar** ‚Äî replaces placeholder
2. **Restore workspace** ‚Äî loads saved folders or CLI workspace/file argument
3. **Restore active file** ‚Äî opens the last active file for fast perceived startup
4. **Wire status bar** ‚Äî cursor tracking, diagnostics callbacks
5. **Create bottom panels** ‚Äî `_create_bottom_panels()`:
   - Creates `AIChatTabs` (replaces placeholder)
   - Creates `TerminalStack` (replaces placeholder)
   - Wires callbacks (workspace folders, current file, terminal focus)
   - Re-applies saved paned positions
6. **Wire maximize callbacks** ‚Äî for AI chat and terminal panels
7. **Open remaining files** ‚Äî one file per `GLib.idle_add` tick to avoid blocking UI
8. **Restore sketch pad** ‚Äî if a `.zen_sketch` file was in the last session
9. **Show welcome screen** ‚Äî if no files are open
10. **Spawn terminal shell** ‚Äî starts shell process in first workspace folder
11. **Print Full UI visible** ‚Äî all panels created and visible
12. **Unlock paned positions** ‚Äî via `GLib.idle_add`
13. **Schedule background init** ‚Äî `GLib.timeout_add(0, _deferred_background_init)`

### Phase 4: Background Init (`_deferred_background_init`)

Non-visible work deferred past the "Full UI visible" metric:

1. **Refresh git status** ‚Äî async git status for tree view decorations
2. **Start file watcher** ‚Äî monitors workspace for external changes
3. **Schedule diagnostics** ‚Äî workspace-wide diagnostics after 2s

## Lazy Imports in EditorView

`editor_view.py` defers several heavy imports to avoid slowing down `_deferred_init_editor()`:

| Import | Cost (warm) | Deferred to |
|--------|-------------|-------------|
| `dev_pad.log_file_activity` | 15.5ms | Each call site (lazy) |
| `Autocomplete` | 15ms | `EditorTab.__init__()` |
| `GutterDiffRenderer` | 13ms | `EditorTab.__init__()` |
| `EditorMinimap` | 3ms | Tab creation |
| `show_input` | ~1ms | Go-to-Line dialog |

## Lazy Properties

These panels are **never created until first accessed**, keeping startup fast even in Phase 3+:

| Property | Class | Trigger |
|----------|-------|---------|
| `diff_view` | `DiffView` | `Ctrl+D` or file save |
| `system_monitor` | `SystemMonitorPanel` | Menu action |
| `dev_pad` | `DevPad` | `Cmd+.` |
| `sketch_pad` | `SketchPad` | `Cmd+Shift+D` |

## Placeholder Pattern

The key optimization pattern used throughout startup:

```python
# In _create_layout() ‚Äî fast placeholder
self.editor_view = Gtk.Box()
self.editor_view.add_css_class("editor")
self.editor_split_paned.set_start_child(self.editor_view)

# In _deferred_init() ‚Äî real widget swap (after first paint)
from editor.editor_view import EditorView
self.editor_view = EditorView()
self.editor_split_paned.set_start_child(self.editor_view)
self._reapply_saved_positions()  # Child swap resets positions
```

This pattern is used for: TreeView, EditorView, AIChatTabs, TerminalView, StatusBar.

## Position Locking

Swapping placeholder children causes GTK to reset paned positions. Zen locks positions during init:

1. **Lock**: after setting initial positions in `_create_layout()`, attach `notify::position` handlers that force-reset to the saved value
2. **Re-apply**: after each child swap, call `_reapply_saved_positions()`
3. **Unlock**: in Phase 3 (`_deferred_init`), disconnect lock handlers via `GLib.idle_add` to allow user resizing

## Font Pre-Registration

Fonts are registered with CoreText/fontconfig **before** `Gtk.init()` to be included in Pango's initial font map. Two approaches were evaluated:

| Approach | Time | Description |
|----------|------|-------------|
| `font_data.py` import | 78-108ms | 3.8MB Python module with base64+zlib data ‚Üí decode ‚Üí temp files |
| Binary `.ttf` files | ~13ms | Direct file read from `src/fonts/resources/` |

The binary approach is used, saving ~95ms. The `.ttf` files were extracted from `font_data.py` and stored in `src/fonts/resources/`. The `_ZEN_FONTS_PREREGISTERED` env var signals `font_manager.py` to skip re-registration.

## Performance Rules

1. **Never import heavy modules at module level** ‚Äî defer to the phase where they're first needed
2. **Use `Gtk.Box` placeholders** ‚Äî swap in real widgets after first paint
3. **One file per idle tick** ‚Äî open remaining files via `GLib.idle_add` to avoid jank
4. **Lock paned positions** ‚Äî prevent GTK from resetting them during child swaps
5. **Defer OS-specific operations** ‚Äî AppKit import, dock icon, font registration
6. **Lazy properties** ‚Äî panels not needed at startup should be `@property` with lazy init
7. **Pre-compile bytecode** ‚Äî `python -m compileall` before launch (`$(COMPILE)` in Makefile)
