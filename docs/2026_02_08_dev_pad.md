Activity tracking panel that shows recent user activities with quick links to resume work.

## Overview

Dev Pad is a side panel that tracks and displays what you've been working on, allowing quick navigation back to previous tasks. It acts as a personal activity log and quick-access hub.

## Toggle

- **Keyboard shortcut:** `Cmd+.` (period)
- **Menu:** View → Toggle Dev Pad

## Display Behavior

Dev Pad opens as a **split view** alongside the editor:

| Scenario | Layout |
|----------|--------|
| Files open | 50/50 split: editor left, Dev Pad right |
| No files open | Dev Pad takes full width |
| Close last file | Dev Pad expands to full width |
| Open file from Dev Pad | Opens on left, Dev Pad stays visible |

Press `Cmd+.` again to close the Dev Pad split.

## Features

### Clickable Links
Each activity with a link target is clickable:
- **Files** → Opens in editor
- **AI chats** → Opens chat tab
- **GitHub PRs** → Opens in browser
- **Repos** → Opens folder

### Search & Filter
- **Search bar:** Filter activities by text (title, description, type)
- **Filter buttons:** Quick filter by type (All, PRs, Notes, Ideas, Reminders)

### Manual Notes
Click **"+ Add Note"** to create:
- 📌 Notes
- 💡 Ideas
- ⏰ Reminders

### GitHub PR Integration
Click **"🔀 Refresh PRs"** to fetch open PRs from workspace repos using `gh` CLI.
PRs are fetched in parallel from all workspace folders.

### Activity Management
- Click **×** on any activity to delete it
- Click **"Clear All"** to clear all (or currently filtered) activities

## Persistence

Activities are stored in `~/.zen_ide/dev_pad.json`.

Data structure:
```json
{
  "activities": [
    {
      "id": "1707412345.123_0",
      "timestamp": "2024-02-08T16:45:00",
      "activity_type": "file_open",
      "title": "main.py",
      "description": "Open ~/project/main.py",
      "link_type": "file",
      "link_target": "/Users/me/project/main.py",
      "metadata": {}
    }
  ]
}
```

Maximum 500 activities are retained (oldest are trimmed).

## Logging Activities Programmatically

Helper functions are provided to log activities from other parts of the app:

```python
from dev_pad import (
    log_file_activity,
    log_ai_activity,
    log_git_activity,
    log_search_activity,
    log_custom_activity,
    log_github_pr_activity,
    log_new_file_activity,
    remove_new_file_activity,
)

# File operations
log_file_activity("/path/to/file.py", action="open")  # or "edit", "save"

# AI interactions
log_ai_activity("How do I parse JSON?", chat_id="chat_123")

# Git operations  
log_git_activity("checkout", branch="feature-x", repo_path="/path/to/repo")

# Searches
log_search_activity("TODO", results_count=42)

# Custom activities
log_custom_activity(
    activity_type="deploy",
    title="Deployed v1.2.3",
    description="Production deployment completed",
    link_type="url",
    link_target="https://example.com/deployments/123"
)

# GitHub PRs
log_github_pr_activity(
    author="octocat",
    title="Fix bug in parser",
    pr_url="https://github.com/org/repo/pull/42",
    repo_name="repo",
    created_at="2024-02-08T10:00:00Z"
)

# New unsaved files (returns activity ID for later removal)
activity_id = log_new_file_activity(tab_id=1)
remove_new_file_activity(activity_id)  # When file is saved or closed
```

### Update vs Add

`update_or_add_activity()` prevents duplicates by matching on `link_type` and `link_target`. If a matching activity exists, it updates the timestamp and moves it to the top instead of creating a duplicate.

## Architecture

| File | Description |
|------|-------------|
| `src/dev_pad/dev_pad.py` | UI panel (GTK4) |
| `src/dev_pad/dev_pad_storage.py` | Storage backend |

The storage uses a singleton pattern (`DevPadStorage.get_instance()`) and supports listeners for real-time UI updates when activities change.

## Theme Support

Dev Pad automatically adapts to the current theme and responds to theme changes via the `subscribe_theme_change()` callback.
