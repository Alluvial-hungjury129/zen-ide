A split diff view for comparing the current file with its commit history.

## Overview

The Diff View provides a side-by-side comparison between your current file (working copy) and any previous commit version. It displays as a full-screen overlay panel over the editor.

**Location:** `src/diff_view.py`

## Features

### Side-by-Side Comparison
- **Left pane:** Commit version (historical)
- **Right pane:** Current version (your changes)
- Synchronized scrolling between both panes
- Syntax highlighting preserved in both editors

### Diff Highlighting
Color-coded backgrounds indicate change types:

| Color | Type | Description |
|-------|------|-------------|
| Green | Addition | Lines added in current version |
| Red | Deletion | Lines removed (present in commit) |
| Orange | Change | Lines modified between versions |
| Purple | Whitespace | Whitespace-only changes (trailing spaces/tabs) |

### Commit Navigation
- Navigate through file's commit history with ← → arrow keys
- Header shows commit info: SHA, date, and message preview
- Click prev/next buttons in header to browse commits

### Revert Individual Changes
- Click the arrow icon in the gutter next to any changed region
- Reverts only that specific change back to the commit version
- Confirmation dialog before applying revert
- File content updates in-place without closing diff view

### Minimap
- Vertical strip on the right showing change locations
- Click to jump to specific changes
- Viewport indicator shows current scroll position
- Color-coded markers match diff highlighting colors

### Whitespace Detection
- Detects whitespace-only changes (trailing spaces, tabs)
- Shows notice banner when only whitespace differences exist
- Purple highlighting distinguishes from actual content changes

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Escape` | Close diff view |
| `←` | Navigate to older commit |
| `→` | Navigate to newer commit |
| `↑` / `↓` | Scroll line by line |
| `Page Up` / `Page Down` | Scroll by page |
| `Home` / `End` | Jump to start/end |
| `Cmd/Ctrl + =` | Zoom in |
| `Cmd/Ctrl + -` | Zoom out |

## Mouse Interactions

| Action | Result |
|--------|--------|
| Scroll wheel | Scroll both panes (right pane leads) |
| Double-click (right pane) | Close diff and navigate to that line |
| Click minimap | Jump to that location (animated) |
| Click revert arrow | Revert that change region |

## Scroll Behavior

- **Right editor scrolling** syncs the left editor (right leads, left follows)
- **Left editor scrolling** is independent (free scroll)
- Smooth animated scrolling for minimap jumps and Home/End

## Opening the Diff View

The diff view is typically triggered via:
- Menu: View → Show Diff with Main Branch
- Keyboard shortcut: `Cmd+Shift+D` (macOS) / `Ctrl+Shift+D` (Windows/Linux)

## Requirements

- File must be in a git repository
- File must have commit history (at least one commit)
- Uses `GitManager` for all git operations

## Technical Details

### Components

| Class | Purpose |
|-------|---------|
| `DiffView` | Main diff view panel and orchestration |
| `DiffMinimap` | Change location overview with click-to-jump |
| `RevertGutterRenderer` | Gutter renderer for per-region revert arrows |

### Diff Algorithm
Uses Python's `difflib.SequenceMatcher` with normalized line comparison (strips trailing whitespace for matching, then detects whitespace-only differences separately).

### Git Integration
- Fetches commits via `GitManager.get_file_commits()`
- Retrieves file content at specific commits via `GitManager.get_file_at_ref()`
- Background thread loading to keep UI responsive

### Theme Support
Fully themeable - colors adapt to the active Zen IDE theme while maintaining diff-specific highlighting colors for clarity.
